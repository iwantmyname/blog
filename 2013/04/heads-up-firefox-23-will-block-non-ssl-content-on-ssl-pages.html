Heads Up: Firefox 23 will block non-SSL content on SSL pages

Web developers, it’s time to check your content! Mozilla has announced that Firefox 23, which is due for release into beta on May 17, 2013 is going to feature a change that blocks all content that is non-SSL on an SSL secured page. Many websites still load elements in an insecure fashion even when their website is SSL secured, so this change is hopefully going to put a swift end to that. There is some disagreement over what developers should be using to solve the problem over on Hacker News, some argue for using protocol-relative URL’s in linking and others argue that linking should explicitly indicate the protocol. Regardless of how you fix it, Firefox 23 will exit beta and hit the stable channel on August 6, 2013. It’s probably time to start planning for this now, not after it happens.
